### 2026-05-28 — Use Agile terminology for project breakdown

Decision:
Use Agile-style terms when discussing and organising project work.

Preferred hierarchy:
Project → Epic → Feature → User Story / Slice → Task → Step.

Usage notes:
- Use "Epic" for major areas of the project.
- Use "Feature" for a clear capability within an epic.
- Use "User Story" when describing functionality from the user's perspective.
- Use "Slice" for a small, testable vertical piece of work that can be completed and verified.
- Use "Task" for the concrete work items needed to complete a story or slice.
- Use "Step" for the next command or action to perform.

Reason:
Don is already familiar with Agile terminology, so using Agile language will make the project structure easier to understand and maintain.

Status:
Accepted.
